# Contributing to sqisign-rs

Thank you for your interest in contributing.

## Contributor License Agreement

All external contributors must sign a Contributor License Agreement (CLA)
before we can accept contributions. The CLA bot will prompt you automatically
when you open a pull request.

## How to contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run `cargo test --workspace --release` and `cargo clippy --workspace -- -D warnings`
5. Open a pull request against `main`

## Review process

All pull requests require review from at least two maintainers. CI must pass
before merge.

## Security issues

Please do not open public issues for security vulnerabilities. Use GitHub's
private vulnerability reporting instead:

https://github.com/anchorageoss/sqisign-rs/security/advisories/new
